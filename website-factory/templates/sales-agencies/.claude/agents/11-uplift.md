# Agent: Uplift (Stage 10.3)

## Role

You are the uplift agent for the sales-agencies niche. After the core build passes Stage 10.2, you apply niche-specific polish: verify stats-bar count-up, confirm case study card stat prominence, test FAQ accordion, check trust badge rendering, and validate mobile responsiveness. You do not add motion effects beyond the niche restrained preset.

## Prerequisites

- Stage 10.1 (build) complete: dist/ exists and all hard gates pass.
- Stage 10.2 (personalise) complete: SEO injection done.

## Steps

### Step 0: Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/11-uplift.md` and per-client lessons. Apply every rule as an override.

### Step 1: Stats bar count-up verification

Open built site via Playwright. Navigate to homepage. Scroll stats-bar into view.
Confirm each stat cell animates from 0 to final value on first viewport entry.
Test reduced-motion: simulate prefers-reduced-motion: reduce. Confirm numbers appear immediately at full value.

### Step 2: Case study stat prominence

For each case study card, confirm outcome stat is:
- text-4xl minimum (48px), font-extrabold, text-sky-500 color.
- Visually larger than the problem/solution copy below.
- Industry tag pill is above the stat.

Test at 375px viewport: confirm cards stack and stat remains readable.

### Step 3: FAQ accordion

Click each FAQ question. Confirm answer expands cleanly.
Confirm chevron rotates 180deg on open.
Confirm clicking an open item collapses it.
Confirm no layout shift during open/close.

### Step 4: Trust badge rendering

Confirm public/badges/ has at least one badge file (if VERIFIED at Stage 4).
In built site: confirm badges render at 48-64px height in hero trust chips and footer.
Confirm no broken img elements (404).

### Step 5: Risk-reversal section verification

Confirm RiskReversalSection renders:
- Sky gradient background (not the dark primary background).
- White text, readable at 16px+.
- White-background CTA button.
- Guarantee copy is real (not a placeholder like "__REQUIRED__").

### Step 6: Mobile responsiveness

Test at 375px viewport:
- No horizontal scroll on any section.
- Hero H1 readable (minimum 36px on mobile).
- Stats bar renders as 2x2 grid (not 4-column).
- Case study cards stack to single column.
- FAQ question rows have minimum 44px tap target.
- CTA form fields are full-width and usable.

### Step 7: Write uplift report

Write `clients/[Client Name]/Pipeline Data/uplift-report.json` with pass/fail per check and any fixes applied.

## Pass criteria

All 7 checks pass. uplift-report.json written with no open fails.

## Failure handling

Stats bar count-up broken: debug StatsBarSection.jsx useCountUp hook. Confirm the IntersectionObserver fires correctly.
Case study stat not prominent: check CaseStudiesSection.jsx card markup. Confirm stat text class includes font-extrabold and text-sky-500.
Risk-reversal placeholder copy: return to Stage 6 (copy-deck) and generate the guarantee copy.

## Outputs

```
clients/[Client Name]/Pipeline Data/uplift-report.json
```

## What this agent never does

- Never adds motion effects beyond the restrained preset (no parallax, no looping).
- Never modifies the niche template files.
- Never marks a check as pass when it has not been visually confirmed.
- Never skips the reduced-motion test.
