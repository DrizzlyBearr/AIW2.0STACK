# 03 - Asset Scraper

Implements: Stage 4.

## Purpose

Harvest the client's brand assets from their website, Google Business Profile, LinkedIn, and linked social channels into the `[Client Name] Assets/` folder. For the sales-agencies niche this means: agency logo (required), founder headshot (required), optional team member photos, client logos for the social proof bar, and trust badges from Clutch and G2.

The asset-scraper halts the entire pipeline if no logo is found. All other categories are optional but flagged as MISSING when absent.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/research/research.json
  - company.url (primary scrape target)
  - social.linkedin
  - certifications[]
clients/[Client Name]/Pipeline Data/intake/intake.json
  - existingSite (boolean)
  - clutchProfileUrl
  - g2ProfileUrl
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/03-asset-scraper.md` and `clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule as an override to this spec.

### Step 2: Logo harvest

Run `tools/apify-scrape.py --target {company.url} --categories logo`.
Match using niche-playbook/asset-patterns.json category `logo`:
- altKeywords: logo, brand, company name, wordmark
- selectorHints: header img, nav img, .logo img, a[href='/'] img
- Require: minimum 200px wide, SVG or PNG.

If logo found: save to `clients/[Client Name] Assets/logo/logo-original.[ext]`.
If SVG found: generate a white-on-transparent variant for dark backgrounds.
If no logo found: HALT. Write asset-report.json with halt reason. Do not proceed to Stage 5.

### Step 3: Founder headshot harvest

Match using asset-patterns.json category `founder`.
Target pages: /about, /team, /our-team, homepage founder section. Also scrape LinkedIn if provided.
Minimum viable: 600x600px. Save to `clients/[Client Name] Assets/team/founder.[ext]`.
If not found: flag as MANUAL-DROP-NEEDED, continue.

### Step 4: Team member photos

Match using asset-patterns.json category `team`. Require minimum 400x400px.
Save to `clients/[Client Name] Assets/team/team-[name].[ext]`.
If none found: flag, continue.

### Step 5: Client logos for social proof bar

Match using asset-patterns.json category `client-logos`.
Look for Clients, Trusted By, or Our Clients section on the agency site.
Require: legible brand name at 32px rendered height. If fewer than 4 found: flag for manual review.

### Step 6: Trust badge verification

Cross-reference research.json certifications against niche-playbook/trust-signals.json badge matchKeywords.
For each matched badge: confirm the client live profile on the issuer site.
Mark as VERIFIED or UNVERIFIED in asset-report.json.
Do NOT fabricate badge image files. Refer to niche-playbook/trust-badges/MANUAL-DROP-NEEDED.md.

### Step 7: Write asset report

Write `clients/[Client Name]/Pipeline Data/asset-report.json` documenting:
- Each category: found/not-found/flagged, file paths, dimensions.
- Trust badges: verified/unverified, issuer URL.
- MANUAL-DROP-NEEDED items with exact filenames.

## Pass criteria

- Logo: found, minimum 200px wide, saved to correct path.
- Asset report: written, valid JSON.
- No fabricated assets. All assets sourced from live URLs or flagged.

## When this stage halts

Logo not found after scraping website, LinkedIn, and all fallback sources.
Write asset-report.json with halt reason. Do not proceed to Stage 5.

## Niche-specific notes

Photo categories in priority order (from photo-manifest.json):
1. logo -- required, halt if missing
2. founder -- required for About page, flag if missing
3. team -- optional, minimum 2 for credible team grid
4. client-logos -- optional, minimum 4 for credible social proof bar
5. trust-badges -- ALL manual drop (Clutch and G2 from live issuer accounts)

Never scrape generic office or meeting room stock photos.
Case-study screenshots: scrape only if a visible results or case-studies page exists on the client site.
