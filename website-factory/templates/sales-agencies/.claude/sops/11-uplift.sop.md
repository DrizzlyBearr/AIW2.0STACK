# 11 - Uplift

Implements: Stage 10.3.

## Purpose

Apply niche-specific polish to the built client site after the core build passes Stage 10.2. For the sales-agencies niche, uplift focuses on: stats-bar count-up animation verification, case study card stat prominence, FAQ accordion smoothness, and trust badge rendering.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/ (built output)
templates/sales-agencies/niche-playbook/motion-preset.json
templates/sales-agencies/niche-playbook/design-vocabulary.md
templates/sales-agencies/.claude/checklists/design-fidelity.md
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/11-uplift.md` and per-client lessons. Apply as override.

### Step 2: Stats bar count-up verification

Open the built site in a browser via Playwright.
Navigate to the homepage. Scroll the stats-bar section into view.
Confirm each of the 4 stat cells animates from 0 to the target number on first viewport entry.
Confirm the animation respects prefers-reduced-motion: disable animation in OS settings, reload, confirm numbers appear at full value immediately.
If count-up is broken: debug StatsBarSection.jsx useCountUp hook.

### Step 3: Case study card stat prominence

For each case study card, confirm:
- Outcome stat renders at minimum text-4xl font-extrabold in text-sky-500.
- Industry tag pill is visible above the stat.
- Problem and solution copy are smaller (text-sm) and below the stat.
- Card layout does not compress on mobile (test at 375px viewport).

### Step 4: Accordion smoothness

Open FAQ section. Click each question.
Confirm answer expands without layout shift.
Confirm chevron rotates 180 degrees on open.
Confirm clicking an open item collapses it.
If accordion is broken: debug FaqSection.jsx open state management.

### Step 5: Trust badge rendering

Confirm public/badges/ contains at least the Clutch badge file (if verified at Stage 4).
In the built site, confirm badges render at ~60px tall in the hero trust chips and footer.
Confirm badges do not render as broken img elements (no 404 badge images).
If badges are missing: check asset-report.json for MANUAL-DROP-NEEDED items and flag to student.

### Step 6: Risk-reversal section check

Confirm RiskReversalSection renders with:
- Sky gradient background (from-sky-600 to-sky-800 or equivalent).
- White text on gradient.
- White-background CTA button.
- Guarantee copy is present and not a placeholder.

### Step 7: Mobile responsiveness check

Test at 375px viewport width:
- No horizontal scroll on any section.
- Hero H1 is readable (minimum 36px on mobile).
- Stats bar renders as 2x2 grid.
- Case study cards stack to single column.
- FAQ accordion is fully tappable (minimum 44px tap target on each question row).

### Step 8: Write uplift report

Write `clients/[Client Name]/Pipeline Data/uplift-report.json` listing:
- Each check: pass/fail.
- Issues found and fixes applied.

## Pass criteria

- All 8 steps pass.
- Uplift report written with no open fails.

## When this stage halts

Any of the following block Stage 10.4a:
- Stats bar count-up broken in both standard and reduced-motion modes.
- Case study card stat is not the largest element on the card.
- Risk-reversal section is missing the guarantee copy.

## Niche-specific notes

The sales-agencies niche uses a restrained motion preset (motion-preset.json).
Do not add motion effects not specified in the preset.
The count-up animation on the stats bar and the accordion are the only JavaScript-driven interactions.
All other animation is CSS transition only.
Do not add: parallax, scroll-driven animation, auto-play video, or looping animations.
