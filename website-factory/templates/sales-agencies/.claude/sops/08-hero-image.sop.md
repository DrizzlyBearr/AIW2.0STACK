# 08 - Hero Image

Implements: Stage 9.

## Purpose

Generate a 1500x700px branded hero image using the Nano Banana skill (Gemini Image API). The hero composition follows the niche playbook's hero-composition.md. Uses the client's logo and optional founder portrait as multimodal reference images.

## Inputs (per-client)

```
clients/[Client Name] Assets/logo/logo-original.[ext]
clients/[Client Name] Assets/team/founder.[ext] (optional)
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
templates/sales-agencies/niche-playbook/hero-composition.md
templates/sales-agencies/niche-playbook/hero-mood-mapping.json
website-factory/.claude/skills/taste/skills/imagegen-frontend-web/SKILL.md
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/08-hero-image.md` and per-client lessons. Apply as override.

### Step 2: Select composition variant

Check whether a founder portrait is available at `clients/[Client Name] Assets/team/founder.[ext]`.
If founder portrait available and >= 600x600px: use Variant B (founder portrait split composition).
If no founder portrait: use Variant A (geometric precision, dark background only).

### Step 3: Select mood

Read brand-dna.json. If `hero.mood` is set, look up in hero-mood-mapping.json moods[].
Default mood: "authority_dark" (matches Variant A).
If using Variant B: switch mood to "founder_portrait".

### Step 4: Construct image prompt

Use the image generation prompt template from hero-composition.md for the selected variant.
Inject client-specific details: brand name, palette primary color, palette accent color.
Append technical spec: "1500 pixels wide, 700 pixels tall, photorealistic quality, no text overlay".

### Step 5: Generate hero via Nano Banana skill

Invoke the nano-banana skill with:
- Prompt from Step 4
- Logo as multimodal reference (for brand color grounding)
- Founder portrait as subject reference (Variant B only)
- Output target: clients/[Client Name]/Pipeline Data/hero-image/hero-final.png

### Step 6: Validate output

Confirm output dimensions: 1500x700px minimum.
Confirm no text in the image (the text overlay is handled by the HTML/CSS layer, not baked into the image).
Confirm no stock photography of generic sales teams or handshakes.
Confirm the image background is predominantly dark (#0F1923 or close) for Variant A.

If output fails validation: re-run with a revised prompt adding constraints. Maximum 2 retry attempts.

### Step 7: Optimise

Run `tools/optimise-image.py --target hero --input hero-final.png`.
Target: quality >= 90, WebP format preferred.

## Pass criteria

- hero-final.png (or .webp) written to hero-image/ folder.
- Dimensions: minimum 1500x700px.
- No text baked into image.
- Variant B: founder is identifiable in the right half of the image.
- Variant A: background is dark, geometric precision aesthetic.

## When this stage halts

- Logo not available (asset-scraper halted). Cannot ground brand colors without the logo.
- Nano Banana API returns an error after 2 retries. Log the error and flag for manual hero replacement.

## Niche-specific notes

For the sales-agencies niche, the hero image is shown on a dark background (#0F1923).
The image must complement, not fight, the text overlay.

What the hero must never show:
- Smiling salespeople shaking hands (stock photography cliche in this category).
- Open-plan office with generic business people.
- Warm sunset tones or golden-hour lighting (wrong register for this niche).
- Any text, logos, or branding baked into the image (handled by CSS layer).

Preferred: Abstract geometric grid at low opacity, clean dark professional palette, or founder portrait with professional studio lighting.
