# Brand DNA Agent

Implements: Stage 7.

## Purpose

Extract the client's brand DNA from their logo, existing website (if any), research data, and copy-deck. Populate the canonical brand-dna.js shape with per-client palette, typography, copy, and metadata. Halt if extraction confidence is below 0.70.

## Inputs (per-client)

```
clients/[Client Name] Assets/logo/logo-original.[ext]
clients/[Client Name]/Pipeline Data/research/research.json
clients/[Client Name]/Pipeline Data/copy/copy-deck.json
clients/[Client Name]/Pipeline Data/strategy/strategy.json
website-factory/.claude/skills/design-synthesis/SKILL.md
references/brand-dna.shape.js
templates/sales-agencies/niche-design-tokens.json (niche defaults)
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/brand-dna-agent.md` and per-client lessons. Apply as override.

### Step 2: Extract palette from logo

Run the design-synthesis skill against the client logo.
For the sales-agencies niche, the niche default palette is: primary #0F1923, accent #0EA5E9.
If the client logo uses a different dominant color:
- Replace `palette.primary` with the client's dominant dark color.
- Replace `palette.accent` with the client's action/brand color.
- Do not change if the client logo is neutral (black, white, gray) -- keep niche defaults.

Populate all 10 palette fields in brand-dna.js from the niche-design-tokens.json as defaults, then override with client-specific colors.

### Step 3: Populate typography

For the sales-agencies niche: typography.heading = "Inter", typography.body = "Inter".
headingFontUrl and bodyFontUrl both use the Inter Google Fonts URL from niche-design-tokens.json.
Override only if the client's existing brand uses a distinctly different heading font (e.g. client site uses Sora for headings). In that case: swap heading to the client font and keep body as Inter.

### Step 4: Populate identity fields

From research.json and strategy.json:
- company.name, company.shortName, company.tagline, company.url
- company.serviceRegion
- contact.phone, contact.phoneTelLink, contact.email
- address.city, address.state
- social.linkedin, social.facebook

### Step 5: Populate copy fields

From copy-deck.json, map all copy.* fields into brand-dna.js.
All copy.buttonText must match copy-locks.json ctaPrimary: "Book a Free Strategy Call".
Do not override locked fields.

### Step 6: Populate niche-specific fields

theme_mode: "dark" (from niche-defaults.json, sales-agencies niche).
shape_motif: "sharp-with-accent-radius".
voice_register: "direct-peer-to-peer-results-led".

### Step 7: Validate and score

Run `scripts/validate-brand-dna.mjs --strict`.
If confidence score < 0.70: halt. Write extraction-report.md with confidence breakdown. Trigger /approve-brand-dna gate.

## Pass criteria

- validate-brand-dna.mjs strict mode passes.
- Confidence >= 0.70.
- No surviving __REQUIRED__ sentinels in brand-dna.js.
- theme_mode = "dark".
- copy.buttonText = "Book a Free Strategy Call".

## When this stage halts

- Confidence < 0.70: write extraction-report.md and halt for /approve-brand-dna.
- Logo not found (asset-scraper halted): cannot extract palette. Do not proceed.

## Niche-specific notes

For the sales-agencies niche, dark theme is mandatory. Do not allow a client override to light mode unless the student explicitly approves it and understands the brand fidelity implications.

The niche uses Inter exclusively. Only override typography if the client's brand guidelines explicitly specify a different heading font. When in doubt, stay with Inter.

Palette primary should be a dark navy or near-black. If the client logo is on a white background and uses a primary brand color that is light (e.g. sky blue as primary), treat the light color as accent and use #0F1923 as primary.
