# 05 - Copy Deck

Implements: Stage 6.

## Purpose

Generate the complete site copy for this client in one consistent voice. Every H1, H2, body paragraph, FAQ answer, case study card, testimonial pull-quote, and CTA label. The copy-deck agent reads the niche playbook copywriting contract and vocabulary before writing a single word.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/strategy/strategy.json
clients/[Client Name]/Pipeline Data/research/research.json
templates/sales-agencies/niche-playbook/copywriting.md
templates/sales-agencies/niche-playbook/vocabulary.json
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/05-copy-deck.md` and per-client lessons. Apply as override.

### Step 2: Load niche copy contracts

Read in full: copywriting.md, vocabulary.json, copy-locks.json, cro-rules.md, copy-blocklist-additions.md.
Internalize the prohibited language list and the voice register before writing.

### Step 3: Build copy manifest

For each section in the wireframe (09-wireframe.md) and each page in the sitemap:
- Read the wireframe annotation for that section.
- Identify which brand-dna.copy.* fields feed that section.
- Plan the copy for each field.

### Step 4: Write hero copy

Apply the heroH1Format from copy-locks.json: "[Audience] get [specific outcome] in [timeframe]."
Populate:
- brandDNA.copy.hero.eyebrow: "Outsourced Sales for [specific audience]"
- brandDNA.copy.hero.headline: H1 (12 words max, 800 weight)
- brandDNA.copy.hero.subheadline: (25 words max, no em-dashes)
- brandDNA.copy.heroTrustChips[]: 4 short claims (from copy-locks.json trustStripClaims, customised with client numbers)

### Step 5: Write stats bar copy

Populate brandDNA.copy.heroTrustChips[] with 4 stat cells using quantified-trust-templates.md.
Replace [N] placeholders with client's actual numbers from research.json.
If numbers are not available: use the niche defaults from copy-locks.json.

### Step 6: Write section copy

For each of the 13 homepage sections, write the copy that fills the brand-dna paths listed in the wireframe.

Key locked fields (from copy-locks.json):
- brandDNA.copy.buttonText = "Book a Free Strategy Call"
- brandDNA.copy.formHeader = "Tell us about your pipeline."
- brandDNA.copy.privacyLine = "No spam. One call, no commitment."
- brandDNA.copy.submitButton = "Book My Free Strategy Call"

Key variable fields (write from research + strategy):
- brandDNA.copy.whyChoose.heading: "Sound familiar?"
- brandDNA.copy.services.heading: short positioning statement for the service section
- brandDNA.copy.cta.heading: the primary conversion pull
- brandDNA.faq[]: 6 questions from buyer-voice research (Sub-task 2)

### Step 7: Write page copy

For each non-homepage page: write a brief set of copy overrides in `copy-deck.json`:
- /services: services page H1 and subheadline
- /case-studies: case studies page H1 and aggregate stats
- /about: founder story para1 and para2
- /contact: contact hero headline

### Step 8: Audit against blocklist

Run `tools/copy-lint.py` against the generated copy-deck.json.
Fix every hit from copy-blocklist-additions.md and the universal ai-vocab-blocklist.md.
No em-dashes. No prohibited terms. All stats are verified numbers from research.

### Step 9: Write output

Write `clients/[Client Name]/Pipeline Data/copy/copy-deck.json` with all populated brand-dna.copy.* fields.

## Pass criteria

- copy-deck.json written, valid JSON.
- Zero prohibited terms (copy-lint.py passes).
- Zero em-dashes in all fields.
- copy.buttonText matches copy-locks.json ctaPrimary exactly.
- copy.formHeader matches copy-locks.json formHeader exactly.
- All heroTrustChips contain real numbers (not placeholders).

## When this stage halts

- copy-lint.py returns more than 3 blocklist hits after 2 revision passes. Flag for manual review.
- Strategy.json missing targetAudience: cannot write niche-specific H1 without knowing the audience.

## Niche-specific notes

Voice register: direct, peer-to-peer, results-not-features.
Never soften the pain in the problem-agitate section. Name it exactly.
Case study cards must lead with the outcome number, not the client name.
Risk-reversal statement must be explicit. See cro-rules.md Rule 5.
The ROI anchor sentence appears in exactly one section per site (not in hero, not in FAQ, not repeated).
