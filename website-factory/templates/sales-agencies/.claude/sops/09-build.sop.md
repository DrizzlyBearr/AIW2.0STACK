# 09 - Build

Implements: Stage 10.1.

## Purpose

Clone the sales-agencies niche template, overlay per-client brand-dna.js values, run the Vite build, and validate the output. The build agent reads the wireframe component order and renders the correct 13-section homepage plus all sub-pages.

## Inputs (per-client)

```
templates/sales-agencies/ (niche template)
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/Pipeline Data/copy/copy-deck.json
clients/[Client Name]/Pipeline Data/strategy/strategy.json
clients/[Client Name]/[Client Name] Assets/ (logos, photos, badges)
research/02-niche-research/sales-agencies/09-wireframe.md
research/02-niche-research/sales-agencies/09-sitemap.json
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/09-build.md` and per-client lessons. Apply as override.

### Step 2: Clone niche template

Run `tools/build-from-template.py --niche sales-agencies --client "[Client Name]"`.
This copies templates/sales-agencies/ into clients/[Client Name]/[Client Name] Website/.
Verifies no GENERATION-FAILED.md marker exists before cloning.

### Step 3: Fill brand-dna.js

Merge brand-dna.json + copy-deck.json + strategy.json into src/config/brand-dna.js.
Run `scripts/validate-brand-dna.mjs` in strict mode.
Fix all sentinel (__REQUIRED__*__) fields that remain.

### Step 4: Copy client assets

Logo: copy to `public/logo/`.
Team photos: copy to `public/team/`.
Trust badges: copy to `public/badges/`.
Client logos: copy to `public/client-logos/`.
Hero image (from Stage 9): copy to `public/hero/hero-final.[ext]`.

### Step 5: Run inject-theme.mjs

Run `scripts/inject-theme.mjs` to rewrite CSS variables, Google Fonts import, html data-theme-mode, title, meta description, and JSON-LD from brand-dna.js.

### Step 6: Confirm component composition order

Verify HomePage.jsx imports 13 sections in the exact wireframe order:
1. HeroSection
2. StatsBarSection
3. ProblemAgitateSection
4. SolutionSection
5. HowItWorksSection
6. CaseStudiesSection
7. ObjectionHandlingSection
8. TestimonialsSection
9. RiskReversalSection
10. PricingAnchorSection
11. FaqSection
12. CtaSection
13. FooterSection

If any section is out of order: fix HomePage.jsx before running the build.

### Step 7: npm install and build

Run `npm install --prefer-offline` in the client website directory.
Run `npm run build`.
Confirm dist/ is produced and non-empty.
Fix any build errors before proceeding.

### Step 8: Validate build output

Run `tools/validate-niche-template.py --niche sales-agencies --validate` pointing at the client build.
All hard gates (1, 2, 3, 6) must pass.
Fix soft-gate warnings (4, 5) where feasible.

## Pass criteria

- dist/ produced after npm run build.
- validate-brand-dna.mjs passes in strict mode (no surviving sentinels).
- validate-niche-template.py hard gates all pass.
- Component order matches wireframe.
- All 13 homepage sections render.

## When this stage halts

- GENERATION-FAILED.md marker exists in templates/sales-agencies/. Re-run /build-niche-template.
- brand-dna.js has surviving sentinel fields after merge. Debug the merge source.
- npm run build fails. Read the Vite error and fix the specific component named.

## Niche-specific notes

Component composition order is critical. The 13 sections must render in wireframe order.
The stats-bar (section 2) directly follows the hero. Do not insert any section between them.
The risk-reversal section (section 9) is always rendered. Never omit it, even if the client has no formal guarantee yet.
CtaSection renders with id="lead-form" -- all hero and section CTAs anchor to this id.
