# 14 - Proposal

Implements: Stage 13.

## Purpose

Generate the final HTML proposal for the client. The proposal walks the agency's new client through every page of their new website, explains what each section does for their buyers, and presents the investment. The proposal-pages.json drives the sitemap pyramid displayed in the proposal modal.

## Inputs (per-client)

```
templates/sales-agencies/niche-playbook/proposal-pages.json
clients/[Client Name]/Pipeline Data/strategy/strategy.json
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/Pipeline Data/deploy/deploy.json (live URL)
website-factory/.claude/skills/proposal-writing/SKILL.md
research/02-niche-research/sales-agencies/agency-profile.md (student agency profile)
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/14-proposal.md` and per-client lessons. Apply as override.

### Step 2: Load niche proposal pages

Read templates/sales-agencies/niche-playbook/proposal-pages.json.
Load all pages[] entries plus applicable aliasPages[].
Alias pages: match strategy.json services[] against aliasPages[].urlFromService needles.
For each matched alias: resolve the URL and include the page in the proposal pyramid.

### Step 3: Build proposal data

From brand-dna.json and strategy.json, extract:
- brandShort: company.shortName
- ownerFirst: team.founder.displayName (first name only)
- serviceCount: length of services[]
- liveUrl: from deploy.json

### Step 4: Render proposal HTML

Run `tools/build-proposal.py --client "[Client Name]" --niche sales-agencies`.
This reads proposal-pages.json, inserts per-client data via {token} substitution, and renders the proposal HTML.

### Step 5: Verify proposal content

Confirm the proposal includes:
- Homepage: all 13 sections explained.
- Services page: present with service names from strategy.json.
- Case Studies page: present, notes that case studies are populated post-launch.
- About and Contact pages: present.
- Alias pages for services matched: outsourced-sales, fractional-sales, or sdr-services as applicable.

### Step 6: Write final proposal

Write `clients/[Client Name]/[Client Name] Proposal/proposal.html`.
Confirm the file is self-contained (no external dependencies beyond the deployed site URL).

## Pass criteria

- proposal.html written and self-contained.
- All pages from proposal-pages.json present in the pyramid.
- Live URL from deploy.json embedded.
- Client name and owner first name substituted (no raw {token} placeholders in output).

## When this stage halts

- deploy.json missing (site not yet deployed). Run Stage 11 (deploy) first.
- tools/build-proposal.py returns an error on proposal-pages.json validation.

## Niche-specific notes

The sales-agencies niche proposal must emphasize:
1. The risk-reversal section (unique to this template) -- explain to the client what it is and why it converts skeptical buyers.
2. The case studies section -- note that it requires 2 minimum case studies to launch with full credibility. Flag if fewer than 2 case studies were provided.
3. The objection-handling section -- explain that it addresses the top 3 fears in the buyer's mind (from Sub-task 2 research).

The proposal voice follows the same direct register as the site copy. Do not use agency-speak in the proposal.
