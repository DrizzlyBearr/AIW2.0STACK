# 04 - Strategy

Implements: Stage 5.

## Purpose

Build the client's service and sitemap strategy. For the sales-agencies niche, strategy means: defining the 2-4 core services the agency offers, the ICP it targets, the geographic service region, and confirming the sitemap from 09-sitemap.json. Output drives Stage 6 copy-deck and Stage 10.1 build.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/research/research.json
  - company.description
  - services[] (from website scrape)
  - serviceAreas[] (from intake or research)
clients/[Client Name]/Pipeline Data/intake/intake.json
  - primaryService
  - targetAudience
  - serviceRegion
  - existingServices[]
```

## Procedure

### Step 1: Read accumulated lessons

Read `.claude/lessons/by-agent/04-strategy.md` and per-client lessons. Apply as override.

### Step 2: Define core services

Review research.json and intake.json to identify 2-4 core services the agency offers.
For sales-agencies niche, valid services are: outsourced SDR, fractional sales leadership, sales playbook build, pipeline management, appointment setting, CRM setup.

Do not invent services not evidenced in research or intake. If the client lists only one service, confirm with the intake form before expanding.

Populate `strategy.json` services[] array with: slug, name, body (1-3 sentences describing the service and who it is for).

### Step 3: Confirm ICP and service region

Extract the target buyer from research.json and intake.json.
For sales-agencies niche, ICP is typically one of: B2B SaaS founders, professional services firms, manufacturing companies, or a named vertical.
Record in strategy.json as `targetAudience` (string) and `serviceRegion` (string, e.g. "United States" or "UK and Europe").

### Step 4: Confirm sitemap

Load niche-playbook at templates/sales-agencies/niche-playbook/proposal-pages.json.
Confirm which pages apply to this client. For most clients the full sitemap applies:
/, /services, /case-studies, /about, /contact.

If the client has no case studies yet: mark /case-studies as PENDING. Stage 10.1 will render placeholder content.

Write `strategy.json` sitemap[] array listing confirmed page slugs.

### Step 5: Define process steps

Load niche-playbook/process.json. Confirm the 3-step Audit / Build / Scale process applies.
If the client uses different terminology (e.g. "Discovery / Deploy / Refine"), update process.json for this client only. Do not change the niche template's canonical process.json.

### Step 6: Write strategy.json

Write `clients/[Client Name]/Pipeline Data/strategy/strategy.json` with:
- services[] -- confirmed services array
- targetAudience -- ICP description
- serviceRegion -- geographic scope
- sitemap[] -- confirmed page slugs
- processSteps[] -- confirmed 3-step process for this client

## Pass criteria

- strategy.json written and valid JSON.
- Minimum 1 service defined.
- targetAudience not empty.
- sitemap contains "/" (homepage) at minimum.

## When this stage halts

- Intake form is incomplete: primaryService or serviceRegion missing. Ask the student to complete intake before proceeding.
- Research data is insufficient to define services confidently.

## Niche-specific notes

Sales-agencies niche service hints:
- Prefer "Assigned SDR Placement" over "Lead Generation" (more specific, more credible).
- Prefer "Sales Playbook Build" over "Sales Consulting" (deliverable, not vague advice).
- Prefer "Outsourced Pipeline Management" over "Sales Outsourcing" (mechanism, not category).

Area hints: sales-agencies typically serve nationwide or multi-country. serviceRegion defaults to the student's country unless the client specifies a narrower territory.
