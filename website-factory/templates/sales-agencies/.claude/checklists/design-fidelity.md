# Design Fidelity Checklist -- sales-agencies niche layer

Generated by Module 2D Phase 10 from the niche wireframe at
`research/02-niche-research/sales-agencies/09-wireframe.md` and the
design specification at `research/02-niche-research/sales-agencies/09-template-spec.md`.

Load order: this per-niche layer is merged with
`website-factory/.claude/checklists/design-fidelity.universal.md` (when present)
by the design-fidelity-qa-agent at Stage 10.4a.

Aggregate target: 0.90 weighted mean across all regions, with no region below
its individual threshold.

---

## NICHE-SPECIFIC REGION THRESHOLDS

Region weights are normalised so the weighted mean target is 0.90.
Sections above the fold or adjacent to the primary CTA carry higher weight.

| Region | Threshold | Weight | Why |
|---|---|---|---|
| Hero | 0.90 | 0.25 | First impression; highest conversion impact |
| Stats Bar | 0.87 | 0.08 | Quantified trust immediately below hero |
| Problem Agitate | 0.86 | 0.07 | Pain resonance; sets up solution narrative |
| Solution | 0.87 | 0.08 | Core service clarity; second-most-read section |
| How It Works | 0.86 | 0.07 | Process credibility; de-risks the buy |
| Case Studies | 0.90 | 0.20 | Proof; highest SSIM weight after hero |
| Objection Handling | 0.85 | 0.06 | Removes final barriers to conversion |
| Testimonials | 0.87 | 0.06 | Social proof; close to the CTA |
| Risk Reversal | 0.88 | 0.04 | Accent strip must render distinctly |
| Pricing Anchor | 0.85 | 0.03 | ROI framing; dark section |
| FAQ | 0.84 | 0.02 | Light section; low conversion weight |
| CTA Section | 0.88 | 0.02 | Form must render correctly |
| Footer | 0.83 | 0.02 | Functional; low visual weight |

Note: weights sum to 1.00. Weighted mean target is 0.90.
No region may score below its stated threshold.

## NICHE-SPECIFIC COMPOSITION CHECKS

### Section order

- [ ] `composition.section_order`, the rendered homepage matches the 13-section
  order defined in `research/02-niche-research/sales-agencies/09-wireframe.md`:
  (1) Hero, (2) Stats Bar, (3) Problem Agitate, (4) Solution, (5) How It Works,
  (6) Case Studies, (7) Objection Handling, (8) Testimonials, (9) Risk Reversal,
  (10) Pricing Anchor, (11) FAQ, (12) CTA Section, (13) Footer.
  Fix: Inspect `src/pages/HomePage.jsx` component order against the wireframe.

- [ ] `composition.above_fold`, the hero section occupies 100vh (full viewport
  height) and the stats bar begins within 20px below the fold on 1280px desktop.
  Fix: Verify `min-h-screen` on HeroSection and remove top margin on StatsBarSection.

### Hero composition

- [ ] `composition.hero.dark_background`, hero background is #0F1923 (not a
  lighter shade or pure black).
  Fix: Restore bg-[#0F1923] on HeroSection root element.

- [ ] `composition.hero.geometric_grid`, the SVG grid overlay renders at 6% opacity
  with 60px x 60px horizontal-and-vertical-only lines in #0EA5E9.
  Fix: Restore the SVG pattern element in HeroSection.jsx.

- [ ] `composition.hero.radial_glow`, a sky-blue radial glow at 4% opacity is
  present centered at the top 30% of the hero.
  Fix: Restore the radial-gradient div in HeroSection.jsx.

- [ ] `composition.hero.no_stock_photography`, hero contains no photographic
  element unless a Variant B founder portrait is explicitly configured in
  brand-dna. Default (Variant A) is text-only.
  Fix: Remove any photographic element from Variant A hero.

- [ ] `composition.hero.text_left_aligned`, headline, subheadline, and CTA row
  are left-aligned (not centered) in the hero text column.
  Fix: Remove text-center class from hero text elements; use text-left.

- [ ] `composition.hero.dual_cta`, both CTA elements render: a primary sky-500
  button linking to #lead-form AND a secondary text link to #case-studies.
  Fix: Restore both CTA elements in HeroSection.jsx.

### Stats bar composition

- [ ] `composition.stats-bar.four_cells`, exactly 4 stat cells render in the
  stats bar row.
  Fix: Verify the stats array slice and StatCell rendering in StatsBarSection.jsx.

- [ ] `composition.stats-bar.dividers`, vertical dividers between cells use
  border-r border-[#1E3A5F] and are hidden on mobile (single-column stacked layout).
  Fix: Restore divider class and responsive hide.

- [ ] `composition.stats-bar.dark_background`, stats bar background is #0F1923
  or equivalent dark surface, not light.
  Fix: Restore dark background class on StatsBarSection.

### Theme alternation

- [ ] `composition.theme_alternation`, the dark/light/dark/light/... alternation
  matches the niche spec:
  dark: hero, stats-bar, problem-agitate, how-it-works, objection-handling,
        pricing-anchor, cta-section, footer.
  light: solution, case-studies, testimonials, faq.
  sky-gradient accent: risk-reversal.
  No two consecutive light sections. No two consecutive dark sections without
  the accent strip between.
  Fix: Inspect section root background classes in each component file.

### Accent (sky-blue) usage

- [ ] `composition.accent_single`, #0EA5E9 (sky-500) is the ONLY accent color.
  No secondary accent color appears anywhere in the template.
  Fix: Remove any non-sky accent color. Decorative variety stays within
  slate/navy family per `niche-playbook/design-vocabulary.md`.

- [ ] `composition.accent_cta_buttons`, all primary CTA buttons use bg-sky-500
  hover:bg-sky-600. No primary button uses a different background color.
  Fix: Restore bg-sky-500 hover:bg-sky-600 on all primary CTA buttons.

### Typography

- [ ] `composition.typography.h1_single`, exactly one H1 element renders on the
  homepage. The hero H1 is the only H1; all other section headings are H2.
  Fix: Audit `src/components/*.jsx` and demote extra H1 elements to H2.

- [ ] `composition.typography.inter_only`, only the Inter font family renders.
  No other typeface is loaded or applied.
  Fix: Remove any non-Inter font import. Verify index.css @import and
  tailwind.config fontFamily.

- [ ] `composition.typography.no_italic_except_allowed`, italic is used only for
  testimonial quotes and objection card headers. Body copy is never italic.
  Fix: Remove italic class from non-testimonial, non-objection body text.

- [ ] `composition.typography.no_center_aligned_body`, body paragraphs are
  left-aligned. Center alignment is reserved for eyebrow labels, section
  headings, and short hero subheadlines only.
  Fix: Remove text-center from paragraph elements; restrict to headings and
  eyebrow labels.

### Shape motif

- [ ] `composition.shape.cards_rounded_lg`, all cards use rounded-lg (8px). No
  rounded-2xl, rounded-3xl, or pill-shaped cards.
  Fix: Replace oversized border-radius on cards with rounded-lg.

- [ ] `composition.shape.buttons_rounded_md`, all buttons use rounded-md (6px).
  Fix: Replace rounded-full or rounded-lg on buttons with rounded-md.

- [ ] `composition.shape.no_glassmorphism`, no glass-morphism (backdrop-blur +
  semi-transparent background) effects are used.
  Fix: Remove backdrop-blur and bg-white/10 from any element.

- [ ] `composition.shape.no_gradient_borders`, no gradient border effects are used.
  Fix: Remove any border-image or gradient-border technique.

### Motion

- [ ] `composition.motion.entrance_only`, only scroll-triggered entrance animations
  are used. No parallax, no auto-play, no looping animations.
  Fix: Remove any parallax or auto-play animation classes.

- [ ] `composition.motion.reduced_motion_guard`, all entrance animations are
  wrapped in a prefers-reduced-motion media query check (useEffect guard or
  CSS @media).
  Fix: Add prefers-reduced-motion guard to all animation-triggering useEffect
  calls and CSS keyframe declarations.

- [ ] `composition.motion.count_up_stats`, stats bar and case study stat numbers
  animate with count-up (not a simple fade). Count-up fires once on first
  intersection.
  Fix: Restore useCountUp hook in StatsBarSection.jsx.

- [ ] `composition.motion.faq_accordion`, FAQ items expand/collapse with a height
  transition (not instant show/hide). Accordion animation is disabled when
  prefers-reduced-motion is active.
  Fix: Restore accordion transition and motion guard in FaqSection.jsx.

### Icons

- [ ] `composition.icons.heroicons_outline`, all decorative icons use Heroicons
  outline style. No filled icons for decorative purposes.
  Fix: Replace any filled icon with its outline equivalent.

- [ ] `composition.icons.stroke_width`, decorative icons use 1.5px stroke;
  functional icons (chevrons, close buttons) use 2px stroke.
  Fix: Set strokeWidth="1.5" on decorative SVGs and strokeWidth="2" on functional.

- [ ] `composition.icons.color_correct`, icons on dark backgrounds use
  text-sky-400; icons on light backgrounds use text-sky-500.
  Fix: Audit icon color class against section background.

### Risk reversal accent strip

- [ ] `composition.risk-reversal.sky_gradient`, the risk-reversal section uses
  the sky gradient (from #0284C7 to #075985) as its background, not a flat color
  and not a dark surface.
  Fix: Restore the inline gradient style on RiskReversalSection.

- [ ] `composition.risk-reversal.white_text`, all text in the risk-reversal
  section is white or near-white (not slate or dark text on the sky gradient).
  Fix: Restore text-white classes on RiskReversalSection text elements.

### Case study cards

- [ ] `composition.case-studies.industry_tag_pill`, each card has an industry tag
  as a rounded-full pill with bg-sky-50 text-sky-700 (light section context).
  Fix: Restore industry tag pill styling in CaseStudyCard.

- [ ] `composition.case-studies.stat_sky_color`, the outcome stat number uses
  text-sky-500 (not white, not slate).
  Fix: Restore text-sky-500 on the outcome stat in CaseStudyCard.

## UNIVERSAL REGRESSION GATES

- [ ] `regression.dist_exists`, `dist/` is produced after `npm run build`.
  Fix: Resolve Vite errors. Check `vite.config.js` and JSX parse errors.

- [ ] `regression.no_404_assets`, every asset reference in the built HTML and JS
  resolves to a real file in `dist/`.
  Fix: Stage 10.1 asset copy + manual drop required for any missing asset.

- [ ] `regression.real_photos`, no placeholder filenames (e.g., `placeholder.jpg`,
  `image_PLACEHOLDER.png`) in `public/work/` or `public/team/`.
  Fix: Stage 4 / Stage 9 / manual drop.

- [ ] `image.hero_quality`, hero images are encoded at quality >= 88 (JPEG or
  WebP). Verified by `tools/optimise-image.py` audit mode.
  Fix: Re-encode with `tools/optimise-image.py --target hero --min-quality 88`.

- [ ] `image.gallery_quality`, gallery and team images are encoded at quality >= 80.
  Fix: Re-encode with `tools/optimise-image.py --min-quality 80`.

  code-split heavy components.

## PASS CRITERIA

1. Every per-niche region SSIM score meets or exceeds its stated threshold.
2. Aggregate weighted mean across all 13 regions meets 0.90.
3. Every composition check passes.
4. Every universal regression gate passes.
5. Every image, mobile, and performance gate passes.

Any region scoring below its threshold is an immediate halt; no override is
permitted without explicit `/override-design-fidelity` command from the student.
