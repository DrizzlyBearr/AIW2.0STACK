## What the Factory does

The Website Factory runs a deterministic, gate-locked, per-client pipeline that clones the active niche template, overlays per-client brand DNA and copy and assets, builds a production Vite + React site, validates it against design fidelity and SOP compliance thresholds, deploys to Vercel, and generates a branded HTML proposal -- all inside the client's own folder under `clients/[Client Name]/`.

---

## Per-niche template

The factory carries no baseline template. Module 2D (`/build-niche-template`) generates a per-niche template from captured niche research and writes it to `templates/{niche-slug}/`. The current niche is `sales-agencies`; its template lives at `templates/sales-agencies/`.

Stage 10.1 (`tools/build-from-template.py`) reads the active niche from the parent stack's `stack-state.json` and clones the niche template into the client folder. If no per-niche template exists for the active niche, Stage 10.1 halts with a clear error pointing back to `/build-niche-template`.

## Inputs the factory needs (per client)

**Minimum required at Stage 1 intake:**
- `businessName` (non-empty string)
- `websiteUrl` (auto-prepends `https://` if missing; must be reachable)

## Brand DNA shape contract (32-key canonical API)

Every component in every niche template reads from `references/brand-dna.shape.js`. Stage 10.1 fills per-client values from `Pipeline Data/brand/brand-dna.json` into the niche template's `src/config/brand-dna.js`. The validator at `scripts/validate-brand-dna.mjs` halts on missing required fields or surviving sentinels.

**Top-level keys in the canonical shape:**

| Key | Type | Notes |
|---|---|---|
| `meta` | object | `title`, `description` -- page meta tags |
| `company` | object | `name`, `shortName`, `tagline`, `url`, `licenseNumber`, `description`, `serviceRegion` |
| `contact` | object | `phone`, `phoneTelLink`, `email`, `googleMapsUrl`, `mapsEmbedUrl` |
| `address` | object | `street`, `city`, `state`, `zip`, `full`, `lat`, `lng` |
| `hours` | object | `weekday`, `saturday`, `display[]`, `emergencyBadge` |
| `businessHours` | object | `tz`, `open`, `close` (24h HH:MM) -- drives "open now" indicator |
| `social` | object | `facebook`, `instagram`, `linkedin`, `youtube` (all nullable) |
| `team` | object | `founder` (name, displayName, title, yearsExp, expLabel), `founders[]` |
| `team_group_photo` | string/null | Resolves to `/team/{filename}` |
| `team_members` | array | `{ filename, name, role }` each |
| `theme_mode` | string | `"light"` / `"dark"` / `"both"` |
| `voice_register` | string | Set by brand-dna-agent from research |
| `shape_motif` | string | Decorative motif slug; resolved against niche asset library |
| `corner_overlay` | object | `{ motif, color, opacity }` -- nullable |
| `palette` | object | 10 colour tokens: `primary`, `primary_dark`, `primary_slate`, `accent`, `accent_light`, `accent_dark`, `neutral`, `neutral_dim`, `silver`, `ink` |
| `typography` | object | `heading`, `body`, `headingFontUrl`, `bodyFontUrl` |
| `reviews` | object | `rating`, `googleCount`, `facebookCount`, `totalReviewCount`, `googleLabel`, `facebookLabel`, `googleStat`, `facebookStat`, `items[]` |
| `services` | array | `{ slug, name, iconPath?, body? }` each |
| `serviceAreas` | array | Array of strings (city names) |
| `trust_badges` | array | `{ filename, alt }` each; resolves to `/badges/{filename}` |
| `press_logos` | array | `{ filename, alt }` each; optional |
| `previous_projects` | array | `{ filename, alt, type?, caption?, category? }` each; resolves to `/work/{filename}` |
| `copy` | object | All page copy bundles (hero, formHeader, buttonText, footerCta, per-section label/heading/body, locked phrases) |
| `process_steps` | array | `{ n, title, body }` each |
| `why_choose_us` | array | Array of strings (titles only) |
| `special_offers` | array | `{ label, description }` each |
| `faq` | array | `{ q, a }` each |
| `blog_posts` | array | Full blog post shape including `slug`, `cover`, `title`, `date`, `category`, `excerpt`, `readTime`, optional `content[]` or `body` |
| `blog_categories` | array | Strings |
| `location_pages` | array | Per-city sub-page objects; empty `[]` for niches without location pages |
| `pages` | object | Per-page copy bundles: `about`, `serviceAreas`, `locationDetail`, `blogPost`, `blog`, `contact`, `services`, `financing` |
| `credit` | object | `{ agency, url }` -- Katleho's agency credit in the footer |

All `__REQUIRED__*__` sentinel values must be replaced by Stage 10.1 before build. The validator halts on any surviving sentinel.

## What the per-niche template and playbook supply

The active niche template at `templates/{niche-slug}/` carries:

- **React components + pages:** 13 components, 6 pages (for `sales-agencies`). The component set and page set are niche-specific -- generated by Module 2D from captured site research and the niche wireframe.
- **Tailwind config + foundation tokens:** Typography scale, palette, spacing, motion presets baked in from niche design tokens.
- **Niche playbook** at `templates/{niche-slug}/niche-playbook/`:

  | Playbook file | Schema | What it controls |
  |---|---|---|
  | `copy-locks.json` | `copy-locks.schema.json` | Locked CTA, form header, privacy line, mobile call label |
  | `trust-signals.json` | `trust-signals.schema.json` | Trust strip count, badge placements, signal priority order |
  | `process.json` | `process.schema.json` | Process section step count and step data |
  | `theme.json` | `theme.schema.json` | Light/dark/both support, palette anchors |
  | `vocabulary.json` | `vocabulary.schema.json` | Approved and banned term lists for the niche |
  | `hero-mood-mapping.json` | `hero-mood-mapping.schema.json` | Hero image prompt construction rules |
  | `motion-preset.json` | `motion-preset.schema.json` | Animation timing and reduced-motion rules |
  | `photo-manifest.json` | `photo-manifest.schema.json` | Required and optional photo asset slots |
  | `asset-patterns.json` | `asset-patterns.schema.json` | Visual asset guidance for the niche |
  | `proposal-pages.json` | `proposal-pages.schema.json` | Proposal page sequence and section config |
  | `copywriting.md` | `copywriting.contract.md` | Voice, tone, niche-specific copy rules |
  | `cro-rules.md` | `cro-rules.contract.md` | Conversion rate optimisation rules for the niche |
  | `design-vocabulary.md` | `design-vocabulary.contract.md` | Visual design language and what to avoid |

- **Per-niche agents and SOPs** at `templates/{niche-slug}/.claude/agents/` and `.claude/sops/`: 10 agents, 8 SOPs specific to the `sales-agencies` niche (copy rules, trust signal placement, hero image direction, proposal page sequence).

## Failure protocol

Any gate failure or loop-cap breach:
1. Write `clients/[slug]/MANUAL-INTERVENTION-NEEDED.md` with exact failures and fix instructions.
2. Set `pipeline-state.json` `status: "halted"`.
3. Append to `build-log.md`.
4. Halt -- no downstream stages run.

---

## Cross-cutting universal invariants (every stage, every niche)

- Zero em-dashes anywhere in output.
- Smart-quote enforcement throughout (curly quotes, real en-dashes, real ellipsis).
- AI-vocab blocklist enforced (loaded from `references/copy/ai-vocab-blocklist.md`).
- `prefers-reduced-motion: reduce` honoured by every animation.
- Schema validity checked at every gate that writes JSON.
- Logo must be present -- pipeline HALTS on missing logo.

## Handoff format

The Module 5 brief lands at `research/output/website-factory-brief.md`. `/tailor-factory` reads it and updates any factory-wide settings that need to align with the niche (e.g. agency branding in the proposal template). The brief is the only artifact Module 5 produces; the factory's pipeline reads from the niche template and playbook directly at runtime.
